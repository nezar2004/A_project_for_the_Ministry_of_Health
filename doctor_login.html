Nabd Doctor Access

Separate secure login for doctors only

Elegant doctor login with direct access to personal bookings.

This page signs the doctor in using Firebase Authentication, then automatically links the account to the doctor profile in Firestore so each doctor can only view their own bookings.

Automatic doctor linking The signed-in account is linked to a doctor profile using email and saved under the doctor record.
Private booking access No doctor can see another doctor’s bookings because the dashboard filters by doctor identity.
Realtime updates New bookings appear instantly using Firebase realtime listeners, not polling every 5 seconds.
Fast workflow Approve, reject, or reschedule from a clean side panel with strong UX and clear information layout.
Use the same email that is linked to the doctor in your Firestore data. After login, the doctor is redirected automatically to doctor_dashboard.html.

Doctor sign in

Enter the doctor email and password to open the secure dashboard.

Best practice:
1. Save each doctor in doctors collection.
2. Store email, doctor_id, name, hospital_name, department_title.
3. Save every patient booking into allBookings with doctor_id and doctor_email.
